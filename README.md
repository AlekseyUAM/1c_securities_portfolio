# Система управления личным портфелем ценных бумаг

Решение для **1С:Предприятие** для управления портфелем финансовых инструментов (акции, облигации и другие ценные бумаги). Позволяет вести учёт, загружать котировки и данные ценных бумаг из API Московской биржи, рассчитывать переоценку и финансовый результат.

## Возможности

- **Справочник ценных бумаг** — ведение НСИ по инструментам, торгуемым на Московской бирже с автоматической загрузкой из MOEX ISS API
- **Загрузка котировок** — получение актуальных цен инструментов
- **Учёт поступления и выбытия** — регистрация покупки/продажи ценных бумаг
- **Переоценка портфеля** — расчёт справедливой стоимости активов по текущим котировкам
- **Начисление НКД** — расчёт накопленного купонного дохода по облигациям
- **Погашение купонов** — учёт выплат купонных доходов
- **Отчёты** — аналитические отчёты по составу и производительности портфеля

## Разработка с использованием Claude Code

Этот проект полностью разработан с использованием **Claude Code 2.1.78** и **агент-ориентированного подхода (Full-Cycle Workflow)**.

### Архитектура разработки

Разработка построена на основе фреймворка [1c-agent-based-dev-framework](https://github.com/SteelMorgan/1c-agent-based-dev-framework), который использует:

- **Полный цикл разработки (Full-Cycle)** — детерминированный процесс с обязательным кросс-ревью на каждом этапе:
  - **Phase 0**: Explorer — анализ кодовой базы, построение графиков зависимостей
  - **Phase 1**: Analyst — создание спецификации (MADR 4.0)
  - **Phase 2**: Architect — проектирование архитектуры и декомпозиция задач (Task Breakdown JSON)
  - **Phase 3a**: Developer-Tests — написание unit-тестов (TDD подход)
  - **Phase 3b**: Developer-Code — реализация функционала по тестам
  - **Phase 4**: Tester — расширение покрытия edge cases и регрессионного тестирования
  - **Reviewer**: Независимая проверка артефактов на каждом этапе

### Тестирование

Проект использует unit-тесты **YaxUnit** с полным покрытием функциональности.
Тесты находятся в расширении `src/cfe/unit_tests/`.
