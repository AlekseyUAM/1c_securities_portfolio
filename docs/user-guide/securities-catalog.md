# Справочник «Ценные бумаги»

## Назначение

Справочник «Ценные бумаги» предназначен для ведения нормативно-справочной информации (НСИ) по финансовым инструментам, торгуемым на Московской бирже. Содержит карточки ценных бумаг с основными идентификаторами, наименованиями и параметрами выпуска.

Справочник поддерживает автоматическую загрузку данных из публичного API Московской биржи (MOEX ISS API), что позволяет минимизировать ручной ввод и обеспечить достоверность информации.

---

## Реквизиты карточки

### Идентификаторы

| Реквизит | Описание | Пример |
|----------|----------|--------|
| **Тикер / SECID** | Биржевой код инструмента | `SBER` |
| **ISIN** | Международный идентификационный код (12 символов) | `RU0009029540` |
| **Номер гос. регистрации** | Государственный регистрационный номер выпуска | `10301481B` |
| **Код** | Внутренний код элемента справочника (заполняется автоматически) | `000000001` |

### Наименования

| Реквизит | Описание | Пример |
|----------|----------|--------|
| **Краткое наименование** | Краткое наименование бумаги (SHORTNAME) | `Сбербанк` |
| **Полное наименование** | Полное официальное наименование | `ПАО Сбербанк` |
| **Наименование (лат.)** | Наименование латиницей | `Sberbank` |

### Параметры выпуска

| Реквизит | Описание | Пример |
|----------|----------|--------|
| **Вид/тип ценной бумаги** | Тип инструмента (акция, облигация и т.д.) | `Акция обыкновенная` |
| **Группа инструмента** | Группа на бирже | `Акции` |
| **Валюта номинала** | Валюта, в которой выражен номинал (ссылка на справочник «Валюты») | `RUB` |
| **Номинальная стоимость** | Номинал одной ценной бумаги | `3.0000` |
| **Объём выпуска** | Количество бумаг в выпуске | `21586948000` |
| **Уровень листинга** | Уровень листинга на бирже (1, 2 или 3) | `1` |
| **Дата начала торгов** | Дата первых торгов инструмента на бирже | `10.06.2009` |

### Служебные

| Реквизит | Описание | Пример |
|----------|----------|--------|
| **Дата загрузки из MOEX** | Дата и время последней загрузки данных из API биржи (только для чтения) | `17.03.2026 14:30:00` |
| **Код эмитента (MOEX)** | Внутренний идентификатор эмитента в системе MOEX | `1199` |
| **Код типа инструмента** | Машиночитаемый код группы (`GROUP`) | `stock_shares` |
| **Код типа бумаги** | Машиночитаемый код типа (`TYPE`) | `common_share` |

## Работа со справочником

### Создание элемента вручную

1. Откройте справочник «Ценные бумаги» из меню подсистемы.
2. Нажмите кнопку **Создать**.
3. Заполните необходимые реквизиты: тикер, ISIN, наименование и другие поля.
4. Нажмите **Записать и закрыть**.

Все реквизиты можно заполнять вручную без обращения к API биржи.

### Загрузка данных из MOEX

Для автоматического заполнения карточки ценной бумаги по данным Московской биржи:

1. Создайте новый элемент справочника или откройте существующий.
2. Заполните хотя бы одно из полей:
   - **Тикер** (например, `SBER`, `GAZP`, `LKOH`) — имеет приоритет при загрузке
   - **ISIN** (например, `RU0009029540`) — используется, если тикер не указан
3. Нажмите кнопку **Загрузить из MOEX** в командной панели формы.
4. При успешной загрузке:
   - Все реквизиты карточки заполнятся данными с биржи
   - Поле «Валюта номинала» заполнится ссылкой на соответствующий элемент справочника «Валюты» (если такой элемент существует)
   - В поле «Дата загрузки из MOEX» зафиксируется текущая дата и время
   - На экране появится уведомление об успешной загрузке
5. Проверьте загруженные данные и нажмите **Записать и закрыть**.

> **Важно:** загрузка заполняет реквизиты формы, но не сохраняет элемент автоматически. После загрузки необходимо записать элемент.

### Повторная загрузка

Если данные по бумаге изменились (например, сменился уровень листинга), можно выполнить повторную загрузку. Все реквизиты обновятся актуальными значениями с биржи. Дата загрузки обновится.

---

## Контроль уникальности

Справочник контролирует уникальность кода **ISIN**: при попытке записать элемент с ISIN, который уже используется другим элементом справочника, будет выведено предупреждение, и запись не выполнится. Это предотвращает дублирование карточек ценных бумаг.

## Возможные ошибки при загрузке

| Сообщение | Причина | Рекомендация |
|-----------|---------|--------------|
| **Нужно заполнить Тикер или ISIN** | Оба поля — тикер и ISIN — пусты | Укажите тикер или ISIN перед загрузкой |
| **Ошибка соединения с MOEX ISS API: ...** | Нет доступа к интернету или сервис MOEX недоступен | Проверьте подключение к интернету и повторите попытку позже |
| **Сервис MOEX вернул ошибку: [код]** | Сервер биржи вернул ошибку (например, 500) | Повторите попытку позже; если ошибка сохраняется, обратитесь к администратору |
| **Ценная бумага "[идентификатор]" не найдена в MOEX** | Бумага с указанным тикером/ISIN не найдена в базе MOEX | Проверьте правильность написания тикера или ISIN |
| **Ценная бумага с ISIN "..." уже существует в справочнике** | Элемент с таким ISIN уже есть в базе | Найдите существующий элемент через поиск по ISIN |

---

## Связь с другими объектами

- **Справочник «Валюты»** — реквизит «Валюта номинала» ссылается на элемент справочника валют. При загрузке из MOEX поиск выполняется по наименованию валюты (символьный ISO-код). Если валюта не найдена в справочнике, реквизит останется пустым — новые валюты автоматически не создаются.

## Примеры использования

### Добавление акции Сбербанка

1. Создайте новый элемент.
2. В поле **Тикер** введите `SBER`.
3. Нажмите **Загрузить из MOEX**.
4. Результат: все поля заполнены — наименование «Сбербанк», ISIN `RU0009029540`, тип «Акция обыкновенная», номинал 3 RUB и т.д.
5. Нажмите **Записать и закрыть**.

### Добавление облигации по ISIN

1. Создайте новый элемент.
2. В поле **ISIN** введите код облигации (например, `RU000A0JXN21`).
3. Нажмите **Загрузить из MOEX**.
4. Результат: реквизиты заполнены данными об облигации.
5. Нажмите **Записать и закрыть**.
