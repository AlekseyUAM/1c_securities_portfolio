# Загрузка котировок ценных бумаг

## Назначение

Обработка «Загрузка котировок ЦБ» предназначена для загрузки исторических цен закрытия ценных бумаг с Московской биржи (MOEX) за указанный период. Загруженные котировки сохраняются в регистре сведений «Котировки ЦБ» и могут использоваться для оценки стоимости портфеля.

Поддерживаются акции, облигации, ETF, депозитарные расписки и паевые фонды, торгуемые на MOEX.

---

## Как загрузить котировки

### Подготовка

Перед загрузкой котировок убедитесь, что в справочнике «Ценные бумаги» заполнены:
- **Тикер** — биржевой код инструмента (например, `SBER`, `GAZP`)
- **Код типа бумаги** — определяет, на каком рынке искать бумагу (акции или облигации)

Бумаги без заполненного тикера не попадут в список для загрузки.

### Пошаговая инструкция

1. Откройте обработку «Загрузка котировок ЦБ».
2. Укажите **Начало периода** и **Окончание периода** — диапазон дат, за который нужно загрузить котировки.
3. В таблице «Список ЦБ» автоматически отобразятся все ценные бумаги из справочника с заполненным тикером. Для каждой бумаги установлен флаг **Загружать**.
4. При необходимости снимите флаг **Загружать** с бумаг, котировки по которым загружать не нужно.
5. Нажмите кнопку **Загрузить**.
6. После завершения загрузки на экране появится сообщение с результатами:
   - Количество обработанных бумаг
   - Количество загруженных котировок
   - Количество бумаг, по которым данные не найдены
   - Количество ошибок (если были)

### Повторная загрузка

При повторной загрузке за тот же период ранее загруженные котировки обновляются актуальными значениями. Дубликаты не создаются.

---

## Настройки

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| **Базовый URL API** | `https://iss.moex.com/iss` | Адрес MOEX ISS API. Изменять не требуется, если используется публичный API биржи |
| **BOARDID акций** | `TQBR` | Режим торгов для акций (основной — Т+) |
| **BOARDID облигаций** | `TQCB` | Режим торгов для облигаций (основной — Т+) |

В большинстве случаев настройки по умолчанию менять не нужно.

---

## Какие данные загружаются

Загружается **цена закрытия** (CLOSE) за каждый торговый день указанного периода. Если цена закрытия недоступна, используется нормативная цена закрытия (LEGALCLOSEPRICE).

Для каждой бумаги автоматически определяется рынок (акции или облигации) на основе типа бумаги из справочника. Если тип бумаги не распознан, система последовательно пробует оба рынка.

При наличии нескольких торговых режимов для одной бумаги берётся цена с основного режима (TQBR для акций, TQCB для облигаций). Если на основном режиме данных нет, используется первый доступный режим с ненулевой ценой.

---

## Просмотр загруженных котировок

Загруженные котировки хранятся в регистре сведений «Котировки ЦБ». Для просмотра:

1. Откройте регистр сведений «Котировки ЦБ» из меню подсистемы.
2. Используйте отбор по ценной бумаге и/или периоду для поиска нужных данных.

Каждая запись содержит:
- **Период** — дата торгов
- **Ценная бумага** — ссылка на элемент справочника
- **Вид котировки** — «Цена закрытия»
- **Значение** — цена в валюте торгов

## Возможные ошибки

| Сообщение | Причина | Рекомендация |
|-----------|---------|--------------|
| **Укажите период загрузки** | Не заполнены даты начала и/или окончания периода | Укажите обе даты перед загрузкой |
| **HTTP ошибка [код]** | Сервер MOEX вернул ошибку | Повторите попытку позже. Код 429 — слишком частые запросы, подождите несколько минут |
| **Ошибка соединения** | Нет доступа к интернету или сервер MOEX недоступен | Проверьте подключение к интернету |

При возникновении ошибки по одной бумаге загрузка остальных бумаг продолжается. Информация обо всех ошибках отображается в итоговом сообщении.

---

## Ограничения

- Загружаются только цены закрытия (не поддерживаются цены открытия, максимум, минимум, объём торгов)
- Поддерживаются только бумаги, торгуемые на Московской бирже
- Загрузка выполняется вручную (автоматическая загрузка по расписанию не реализована)
- Для загрузки необходим доступ в интернет с сервера 1С
- Котировки облигаций загружаются в процентах от номинала (как возвращает MOEX API)

---

## Связь с другими объектами

- **Справочник «Ценные бумаги»** — источник списка бумаг для загрузки. Используются реквизиты Тикер и Код типа бумаги
- **Регистр сведений «Котировки ЦБ»** — хранилище загруженных котировок
- **Перечисление «Виды котировок ЦБ»** — определяет тип котировки (на данный момент только «Цена закрытия»)
